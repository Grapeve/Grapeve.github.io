### 3. Rust的编译期计算

编译期计算：编译期函数求值（CTFE），编译时执行的代码，必须知道其结果，才能继续执行。

rust支持的两种编译期计算方式：

- 过程宏 + Build 脚本
  
- 类似于Cpp中的编译期常量表达式求值。
  

**Rust CRFE**

1. 常量函数
  
  `const X:T = ...;` `=`后面的表达式必须是可以在编译期间执行的代码（即常量表达式），`const`关键字创建了一个常量上下文。
  
  常量上下文：
  
  - 常量值初始化位置
    
  - 静态数组的长度表达式，`[T, N]`;
    
  - 重复的长度表达式，类似于 `[0, N]`;
    
  - 静态变量、枚举判别式的初始化位置。
    
  
  常量上下文可接受的常量表达式：
  
  - const fn 函数
  - 元组结构体
  - 元组的值

**3.闭包**

唯一不可变引用：编译器不允许你去使用被闭包捕获的引用。

<span style="color:blue">**闭包实现Copy / Clone 的两条简单规则:**</span>

1.若环境变量实现了Copy，闭包如果以**可变借用方式**捕获环境变量并对其进行修改，则闭包

自身不会实现Copy。

2.若环境变量自身是Move 语义，如果闭包内**捕获环境变量的操作涉及修改环境变量或者消耗环境变量**，则闭包自身不会实现Copy。

<span style="color:blue">**闭包实现Sync / Send的三条简单规则：**</span>

1.如果所有捕获变量均实现了Sync，则闭包女现Sync。

2.如果环境变量**都不是唯一不可变引用方式捕获**的，**并且都实现了Sync**，则闭包实现Send。

3.如果环境变量**是以唯一不可变引用**、**可变引用**、**Copy或Move所有权捕获**的，那闭包实现Send。
